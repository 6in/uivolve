## 4. 原因と恒久対処

原因は注文検索クエリのフルスキャン化。調査クエリと修正差分をタブで併記する。

```uivolve height=340
xtype: tabpanel
itemId: fixTabs
items:
  - title: 調査クエリ
    itemId: investigateTab
    iconCls: x-fa fa-magnifying-glass
    layout: fit
    items:
      - xtype: codeeditor
        itemId: explainSql
        language: sql
        theme: light
        readOnly: true
        value: |
          EXPLAIN ANALYZE
          SELECT o.id, o.no, c.name
          FROM orders o
          JOIN customers c ON c.id = o.customer_id
          WHERE date(o.created_at) = current_date   -- ← 関数適用でインデックスが効かない
          ORDER BY o.created_at DESC
          LIMIT 50;
  - title: 修正差分
    itemId: diffTab
    iconCls: x-fa fa-code-compare
    layout: fit
    items:
      - xtype: diffeditor
        itemId: queryFix
        language: sql
        original: |
          SELECT o.id, o.no, c.name
          FROM orders o
          JOIN customers c ON c.id = o.customer_id
          WHERE date(o.created_at) = current_date
          ORDER BY o.created_at DESC
          LIMIT 50;
        value: |
          SELECT o.id, o.no, c.name
          FROM orders o
          JOIN customers c ON c.id = o.customer_id
          WHERE o.created_at >= current_date
            AND o.created_at < current_date + interval '1 day'
          ORDER BY o.created_at DESC
          LIMIT 50;
```
